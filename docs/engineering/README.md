# Engineering System

Architecture is useful only when it changes everyday engineering behavior. This directory defines the delivery loop and quality guardrails for the AliExpress Clone.

## Start Here

1. Follow the [engineering delivery loop](development-loop.md) for every change.
2. Apply the [testing strategy](testing-strategy.md) before requesting review.
3. Apply the [security baseline](security-baseline.md) to all code and configuration changes.
4. Use [reliability and operations](reliability-and-operations.md) for customer-impacting paths.
5. Meet the [delivery quality gates](delivery-quality-gates.md) before merge or release.

## Principle

Do not claim staff/principal-level quality because a document exists. Produce evidence: passing tests, reviewed contracts, observable behavior, secure configuration, safe rollout, and a learning loop after incidents.
